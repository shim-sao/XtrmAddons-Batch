MySQLBatchBackup

MySQLBatchBackup est une série de scripts permettant de sauvegarder une base de données distante UNIX sur un serveur en local WINDOWS avec les protocoles SHH et SFTP.

1 - Prérequis

  • Avoir Putty d'installé.
  • Télécharger 7-Zip ou l'avoir d'installé.

Optionnel :

  • Avoir MySQL 5+ d'installé.

2 - Configurer l'application

  • Ouvrir le fichier application.cmd avec un éditeur de texte.
  • Vérifier et changer le chemin du dossier MBB_PATH_ZIP où se trouve l'installation du logiciel 7-Zip ou placer l'executable dans le répertoire nommé "zip".
    Commenter l'ajout à la variable d'environnement Windows PATH si un autre chemin de dossier existe déjà.
  • Décommenter et vérifier le chemin du dossier où se trouve l'installation du logiciel Putty si celui ci ne se trouve pas déjà dans la variable d'environement Windows PATH.
  • Décommenter et vérifier le chemin du dossier où se trouve l'installation du logiciel MySQL 5+ si celui ci ne se trouve pas déjà dans la variable d'environement Windows PATH.
  • Sauvegarder.

3 - Configurer un backup ?

Configurer les serveurs et base de données :

  • Dans le dossier config Copier/Coller le fichier config.example.cmd et le renommer en par exemple : config.monsite.cmd
  • Ouvrir le fichier config.monsite.cmd avec un éditeur de texte et éditer les informations suivantes :
    dbSrc = serveur distant. Exemple : 1&1
    dbDest = serveur local.
  • Sauvegarder.

Créer un fichier de lancement :

  • Copier/Coller le fichier backup-example.bat et le renommer en par exemple : backup-monsite.bat
  • Ouvrir le fichier backup-monsite.bat avec un éditeur de texte.
  • Editer les lignes suivantes :
  • MBB_CONFIG_FILE : Remplacer le nom du fichier par celui créé précédemment.
  • MBB_PARAM_BACKUP : Activer ou non l'import automatique dans la base de donnée locale (yes|no).
    Noter que pour cela MySQL 5+ doit être disponible.
  • Sauvegarder.

4 - Lancer le backup

  • Double cliquer sur backup-monsite.bat et suivre les instructions.
